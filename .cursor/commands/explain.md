# Explain Code

Provide a detailed explanation of specified code.

## What to Explain

The user can specify:
- A file path
- A function name
- A module
- A specific line range

## Explanation Format

### Overview
Brief summary of what the code does and why it exists.

### Flow
Step-by-step walkthrough:
1. Entry point
2. Key operations
3. Return value/side effects

### Key Concepts
Rust-specific patterns used:
- Ownership/borrowing
- Async/await patterns
- Error handling approach
- Trait implementations

### Dependencies
- Other modules called
- External crates used
- System resources accessed

### Edge Cases
- How errors are handled
- Boundary conditions
- Special input handling

## Style

- Use simple language
- Include code snippets for context
- Explain the "why" not just "what"
- Note any potential improvements
